# NixOS Configuration

Personal NixOS configuration for multiple machines.

## Philosophy

Keep everything simple.

- No Flakes
- No Home Manager
- No dotfiles management
- No feature/module abstraction
- System config stays in `/etc/nixos`
- User config stays in `~/.config`
- Docker Compose files stay in `/data/compose`

## Structure

```text
/etc/nixos
├── configuration.nix
├── hardware-configuration.nix
├── machine.nix
├── nas.nix
├── thinkpad.nix
├── .gitignore
└── README.md
````

## Files

### configuration.nix

Shared configuration for all machines:

* bootloader
* user account
* desktop environment
* fonts
* packages
* Docker
* Tailscale
* SSH
* common services

### hardware-configuration.nix

Local hardware file generated by NixOS.

This file is not tracked by Git.

It contains machine-specific hardware facts:

* root filesystem
* EFI partition
* CPU microcode
* kernel modules

### machine.nix

Local machine selector.

This file is not tracked by Git.

On NAS:

```bash
cd /etc/nixos
ln -sf nas.nix machine.nix
```

On ThinkPad:

```bash
cd /etc/nixos
ln -sf thinkpad.nix machine.nix
```

`configuration.nix` imports `machine.nix`, and `machine.nix` points to the current machine profile.

### nas.nix

NAS-specific configuration:

* hostname
* `/data` SSD mount
* `nas` user group
* `/home/weis/*` links to `/data/*`
* media compose service
* immich compose service
* rclone compose service
* NAS firewall ports

### thinkpad.nix

ThinkPad-specific configuration:

* hostname
* power management
* laptop-specific settings

ThinkPad does not use `/data`.

It uses normal home directories:

```text
~/Documents
~/Downloads
~/Pictures
~/Music
~/Videos
```

## .gitignore

Local files are ignored:

```gitignore
hardware-configuration.nix
machine.nix
result
```

Reason:

* `hardware-configuration.nix` is machine-specific
* `machine.nix` is a local selector
* `result` is generated by `nixos-rebuild build`

These files should not be pushed or pulled between machines.

## SSH Key

GitHub push/pull uses SSH.

Check existing keys:

```bash
ls ~/.ssh
```

Generate a new key if needed:

```bash
ssh-keygen -t ed25519 -C "songweicn@gmail.com"
```

Show public key:

```bash
cat ~/.ssh/id_ed25519.pub
```

Add it to GitHub:

```text
GitHub → Settings → SSH and GPG keys → New SSH key
```

Test:

```bash
ssh -T git@github.com
```

Use SSH remote:

```bash
git remote -v
```

Expected format:

```text
git@github.com:songweicn/<repo>.git
```

Change remote if needed:

```bash
git remote set-url origin git@github.com:songweicn/<repo>.git
```

## NAS Data Layout

NAS stores real data under `/data`.

```text
/data
├── documents
├── downloads
├── pictures
├── music
├── videos
├── compose
└── container-data
```

Home folders are symlinked:

```text
/home/weis/Documents -> /data/documents
/home/weis/Downloads -> /data/downloads
/home/weis/Pictures  -> /data/pictures
/home/weis/Music     -> /data/music
/home/weis/Videos    -> /data/videos
```

## Verify NAS Data Mount

Before starting Compose services on NAS, confirm `/data` is really mounted.

```bash
findmnt /data
```

Expected:

```text
/data
```

Also check filesystem:

```bash
df -h /data
lsblk -f
```

Expected:

```text
/data mounted from DATA disk
```

If `/data` is not mounted, do not start Compose services.

## Docker Compose

Compose files are intentionally outside `/etc/nixos`.

```text
/data/compose
├── immich
├── media
└── rclone
```

Start manually:

```bash
cd /data/compose/media
docker compose up -d

cd /data/compose/immich
docker compose up -d

cd /data/compose/rclone
docker compose up -d
```

Stop manually:

```bash
cd /data/compose/media
docker compose down
```

Check containers:

```bash
docker ps
```

Check logs:

```bash
docker logs seerr --tail=100
docker logs jellyfin --tail=100
docker logs immich_server --tail=100
```

## Rebuild

Build only:

```bash
sudo nixos-rebuild build
```

Apply:

```bash
sudo nixos-rebuild switch
```

Reboot:

```bash
sudo reboot
```

## Git Workflow

```bash
cd /etc/nixos

git status
git add -A
git commit -m "update nixos config"
git push
```

Do not push machine-specific local files.

## Restore

Fresh NixOS installation:

```bash
sudo mv /etc/nixos /etc/nixos.bak
git clone git@github.com:songweicn/<repo>.git /etc/nixos
cd /etc/nixos
ln -sf nas.nix machine.nix
sudo nixos-rebuild switch
```

For ThinkPad:

```bash
cd /etc/nixos
ln -sf thinkpad.nix machine.nix
sudo nixos-rebuild switch
```

````

保存后：

```bash
cd /etc/nixos
hx README.md
````

粘贴覆盖，然后：

```bash
git add README.md
git status
```
